Haunted House Escape Game

ADDENDUM

Game Design:

I structured the game as a text-based adventure with multiple levels, each presenting the player with choices that lead to different outcomes. The storyline revolves around escaping a haunted house, with various spooky scenarios and twists to keep the player engaged.

HTML Structure:

I organized the HTML file to have separate sections for the start screen, game screen, and footer. This separation helps maintain clarity and allows for easy manipulation of content through JavaScript.

CSS Styling:

The CSS file was designed to give the game a dark, eerie aesthetic, fitting the theme of a haunted house. I utilized responsive design principles to ensure the game looks good on different screen sizes.

JavaScript Functionality:

JavaScript was used to handle the game logic, including rendering levels, processing player choices, and managing game state.

I implemented a dynamic rendering system to display text, choices, and images based on the current game level.

Endgame scenarios were handled to display appropriate messages (either success or failure) and offer the option to restart the game.

Challenges and Solutions:

One challenge was managing the complexity of the game's branching storyline. I addressed this by carefully structuring the level data and using conditional rendering to handle different outcomes.

Ensuring smooth transitions between levels and maintaining a coherent narrative flow required meticulous testing and debugging.

Integrating dynamic content such as the last modified date in the footer involved understanding DOM manipulation and JavaScript's Date object.

Insights Gained:

Developing this game reinforced the importance of clear code organization and separation of concerns, especially in projects with multiple interconnected components.

I gained a deeper understanding of JavaScript event handling, DOM manipulation, and CSS styling techniques.

Testing and iteration were crucial for refining the player experience and identifying and fixing any bugs or inconsistencies.
